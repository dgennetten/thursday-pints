## Architecture

**Thursday Pints** is a read-only React/TypeScript SPA that tracks brewery visits for a social club.

### Data Flow

```
scripts/spreadsheet.xlsx → extract-data.js → public/data.json
public/breweries.json (brewery locations, coordinates)
         ↓
App.tsx loads both JSON files at startup
         ↓
utils.ts: processVisits() + mergeBreweryStatsWithAllBreweries()
         ↓
Three view modes: Tour (chronological) | Ranked (by visit count) | Breweries (by last visit)
Optional map toggle: Leaflet map alongside the list
```

### Key Files

- **`src/App.tsx`** — All application state lives here. Loads data, applies filters/search, and passes props to every component.
- **`src/utils.ts`** — Core data processing: `processVisits()` aggregates raw visit records into brewery stats; `mergeBreweryStatsWithAllBreweries()` joins visit stats with the breweries.json location data.
- **`src/types.ts`** — Shared TypeScript interfaces (`Visit`, `BreweryStats`, `BreweryLocation`, etc.).
- **`src/services/breweryService.ts`** — Fetches `public/breweries.json`.
- **`src/services/spreadsheetService.ts`** — Fetches `public/data.json`; also has legacy OneDrive/XLSX parsing code with localStorage caching.
- **`public/data.json`** — Array of visit records `{ date, breweryName, nextBrewery?, isClosed?, notes? }`. Generated by `extract-data.js`; edit this file directly to add/update visits.
- **`public/breweries.json`** — Array of brewery location records `{ brewery_name, brewery_address, latitude, longitude, status }`. Maintained manually.

### Adding a Visit

The canonical way is to edit `public/data.json` directly (append a new entry with `date` in `YYYY-MM-DD` format and `breweryName`). The `scripts/add-visit.js` CLI and `scripts/extract-data.js` Excel workflow are alternative paths.

### Deployment

GitHub Actions (`.github/workflows/deploy.yml`) auto-deploys on push to main via FTP to Dreamhost. Secrets `FTP_SERVER`, `FTP_USERNAME`, `FTP_PASSWORD` are set in the repo.
